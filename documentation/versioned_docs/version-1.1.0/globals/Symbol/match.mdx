# Symbol.match

The **`Symbol.match`** well-known symbol specifies the matching of a regular expression against a string. This function is called by the [`String.prototype.match()`](../../globals/String/prototype/match.md) method.

For more information, see `RegExp.prototype[Symbol.match]()` and [`String.prototype.match()`](../../globals/String/prototype/match.md).

## Value

The well-known symbol `Symbolmatch`.

## Description

This function is also used to identify [if objects have the behavior of regular expressions](https://developer.mozilla.org/docs/Web/JavaScript/Reference/Global_Objects/RegExp#special_handling_for_regexes). For example, the methods [`String.prototype.startsWith()`](../../globals/String/prototype/startsWith.md), [`String.prototype.endsWith()`](../../globals/String/prototype/endsWith.md) and [`String.prototype.includes()`](../../globals/String/prototype/includes.md), check if their first argument is a regular expression and will throw a [`TypeError`](../../globals/TypeError/TypeError.md) if they are. Now, if the `match` symbol is set to `false` (or a [Falsy](https://developer.mozilla.org/docs/Glossary/Falsy) value except `undefined`), it indicates that the object is not intended to be used as a regular expression object.
